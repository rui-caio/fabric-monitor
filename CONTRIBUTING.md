# Contributing to Fabric Monitor

First off, thank you for considering contributing to Fabric Monitor! It's people like you who make this tool better for everyone.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

## How Can I Contribute?

### Reporting Bugs

- **Search first**: Check if the bug has already been reported in the [Issues](https://github.com/rui-caio/fabric-monitor/issues) tab.
- **Use the template**: If you find a new bug, please use the Bug Report template to provide as much detail as possible.
- **Be specific**: Include your environment details, the behavior you expected, and what actually happened.

### Suggesting Enhancements

- **Open an issue**: Use the Feature Request template to describe the enhancement.
- **Explain the "why"**: Why would this be useful? Who is it for?

### Pull Requests

1. **Fork the repository** and create your branch from `main`.
2. **Install dependencies** (`pip install msal`).
3. **Make your changes**. If you've added code that should be tested, add tests.
4. **Ensure the code style** matches the rest of the project.
5. **Update documentation** (like the README) if needed.
6. **Submit a Pull Request** with a clear description of what you've done.

## Development Setup

The project is a Python backend with a Vanilla JS/HTML frontend.

1. Clone your fork.
2. Create a `.env` file based on `.env.example`.
3. Run `python fabric_proxy.py`.
4. Access `http://localhost:8765`.

## Any Questions?

Feel free to open an issue for questions or join the conversation in existing issues!
